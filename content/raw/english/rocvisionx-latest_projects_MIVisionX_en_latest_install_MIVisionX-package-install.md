  <section id="install-mivisionx-using-the-linux-package-installer">
<h1>Install MIVisionX using the Linux package installer<a class="headerlink" href="#install-mivisionx-using-the-linux-package-installer" title="Link to this heading">#</a></h1>
<p>Three MIVisionX packages are available on Linux:</p>
<div class="line-block">
<div class="line"><code class="docutils literal notranslate"><span class="pre">mivisionx</span></code>: The MIVisionX runtime package. This is the basic rocAL package that only provides dynamic libraries. It must always be installed.</div>
<div class="line"><code class="docutils literal notranslate"><span class="pre">mivisionx-dev</span></code>: The MIVisionX development package. This package installs a full suite of libraries, header files, and samples. This package needs to be installed to use samples.</div>
<div class="line"><code class="docutils literal notranslate"><span class="pre">mivisionx-test</span></code>: A test package that provides a CTest to verify the installation.</div>
</div>
<p>All the required prerequisites are installed when the package installation method is used.</p>
<div class="admonition note">
<p class="admonition-title">Note</p>
<div class="line-block">
<div class="line">The package installation only supports the HIP backend. <a class="reference internal" href="MIVisionX-linux-build-and-install.html"><span class="doc">Build and install from source</span></a> to use the OpenCL backend.</div>
<div class="line"><br /></div>
<div class="line">The FFmpeg and OpenCV dev packages must be installed manually on RHEL and SLES.</div>
</div>
</div>
<section id="basic-installation">
<h2>Basic installation<a class="headerlink" href="#basic-installation" title="Link to this heading">#</a></h2>
