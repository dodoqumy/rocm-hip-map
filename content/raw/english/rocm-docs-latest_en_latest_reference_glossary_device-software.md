This section provides brief definitions of software abstractions and programming models that run on AMD GPUs.

AMDGPU assembly[\#](#term-AMDGPU-assembly "Link to this term"){.headerlink}

:   AMDGPU assembly (GFX ISA) is the low-level assembly format for programs running on AMD GPUs, generated by the [[ROCm compiler toolchain]{.xref .std .std-term}](host-software.html#term-HIP-compiler){.reference .internal}. See [AMDGPU assembly](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/compilers.html#amdgpu-assembly "(in HIP Documentation v7.2.53211)"){.reference .external} for instruction set details.

AMDGPU intermediate representation[\#](#term-AMDGPU-intermediate-representation "Link to this term"){.headerlink}

:   AMDGPU IR is an intermediate representation for GPU code, serving as a virtual instruction set between high-level languages and [[architecture-specific assembly]{.xref .std .std-term}](#term-AMDGPU-assembly){.reference .internal}. See [AMDGPU intermediate representation](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/compilers.html#amdgpu-ir "(in HIP Documentation v7.2.53211)"){.reference .external} for compilation details.

Global memory[\#](#term-Global-memory "Link to this term"){.headerlink}

:   Global memory is the [[device-wide memory]{.xref .std .std-term}](device-hardware.html#term-GPU-RAM-VRAM){.reference .internal} accessible to all [[threads]{.xref .std .std-term}](device-hardware.html#term-Work-item-Thread){.reference .internal}, physically implemented as HBM or GDDR. See [Memory model](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#memory-hierarchy "(in HIP Documentation v7.2.53211)"){.reference .external} for global memory details.

Grid[\#](#term-Grid "Link to this term"){.headerlink}

:   A grid represents the collection of all [[work-groups]{.xref .std .std-term}](device-hardware.html#term-Work-group-Block){.reference .internal} executing a single [[kernel]{.xref .std .std-term}](#term-HIP-kernel){.reference .internal} across the entire GPU. See [Grid](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#inherent-thread-hierarchy-grid "(in HIP Documentation v7.2.53211)"){.reference .external} for grid execution details.

HIP kernel[\#](#term-HIP-kernel "Link to this term"){.headerlink}

:   A HIP kernel is the unit of GPU code that executes in parallel across many [[threads]{.xref .std .std-term}](device-hardware.html#term-Work-item-Thread){.reference .internal}, distributed across the GPU's [[compute units]{.xref .std .std-term}](device-hardware.html#term-Compute-units){.reference .internal}. See [Device programming](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#device-program "(in HIP Documentation v7.2.53211)"){.reference .external} for kernel programming details.

HIP memory hierarchy[\#](#term-HIP-memory-hierarchy "Link to this term"){.headerlink}

:   The memory hierarchy pairs each [[thread hierarchy]{.xref .std .std-term}](#term-HIP-thread-hierarchy){.reference .internal} level with corresponding memory scopes, from [[private registers]{.xref .std .std-term}](https://rocm.docs.amd.com/projects/composable_kernel/en/latest/reference/Composable-Kernel-Glossary.html#term-register "(in Composable Kernel Documentation v1.2.0)"){.reference .external} to [[LDS]{.xref .std .std-term}](device-hardware.html#term-Local-data-share){.reference .internal} to [[GPU RAM]{.xref .std .std-term}](device-hardware.html#term-GPU-RAM-VRAM){.reference .internal}. See [Memory model](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#memory-hierarchy "(in HIP Documentation v7.2.53211)"){.reference .external} for memory architecture details.

HIP thread hierarchy[\#](#term-HIP-thread-hierarchy "Link to this term"){.headerlink}

:   The thread hierarchy structures parallel work from individual [[threads]{.xref .std .std-term}](device-hardware.html#term-Work-item-Thread){.reference .internal} to [[blocks]{.xref .std .std-term}](device-hardware.html#term-Work-group-Block){.reference .internal} to [[grids]{.xref .std .std-term}](#term-Grid){.reference .internal}, mapping onto hardware from [[SIMD lanes]{.xref .std .std-term}](device-hardware.html#term-SIMD-core){.reference .internal} to [[compute units]{.xref .std .std-term}](device-hardware.html#term-Compute-units){.reference .internal} to the entire GPU. See [Hierarchical thread model](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#inherent-thread-model "(in HIP Documentation v7.2.53211)"){.reference .external} for complete details.

LLVM target name[\#](#term-LLVM-target-name "Link to this term"){.headerlink}

:   The LLVM target name is a string identifier corresponding to a specific [[GFX IP]{.xref .std .std-term}](device-hardware.html#term-GFX-IP){.reference .internal} version that is passed to the [[HIP compiler]{.xref .std .std-term}](host-software.html#term-HIP-compiler){.reference .internal} toolchain to specify the target GPU architecture for code generation. See [ROCm compiler reference](https://rocm.docs.amd.com/projects/llvm-project/en/latest/reference/rocmcc.html "(in llvm-project Documentation v22.0.0)"){.reference .external} for details.

ROCm programming model[\#](#term-ROCm-programming-model "Link to this term"){.headerlink}

:   The ROCm programming model defines how AMD GPUs execute massively parallel programs using hierarchical [[work-groups]{.xref .std .std-term}](device-hardware.html#term-Work-group-Block){.reference .internal}, memory scopes, and barrier synchronization. See [Introduction to the HIP programming model](https://rocm.docs.amd.com/projects/HIP/en/latest/understand/programming_model.html#programming-model "(in HIP Documentation v7.2.53211)"){.reference .external} for complete details.
